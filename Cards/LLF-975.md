how pager duty alert gets triggered in `mc-ala`
**How the alert fires — the pipeline:**

1. **[Log.error(...)](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) in [zendesk_service.py](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)** — When both attempts to create a Zendesk ticket fail, [Log.error(retry_response.message)](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html) is called. This emits a log entry with `severity=ERROR` to GCP Cloud Logging, tagged with the Cloud Function's resource labels.
    
2. **Log-based metric** (defined in [main.tf:187](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)) — A `google_logging_metric` resource named `AlaErrorsGenerateFranchiseList` watches for logs matching this filter:
    ```
    resource.type="cloud_function"
	resource.labels.function_name="generate_franchise_list"
	resource.labels.region="australia-southeast1"
	severity=ERROR
    ```
    Every time a matching log line appears, GCP increments this metric counter.
    
3. **Alert policy** ([main.tf:200](vscode-file://vscode-app/Applications/Visual%20Studio%20Code.app/Contents/Resources/app/out/vs/code/electron-browser/workbench/workbench.html)) — A `google_monitoring_alert_policy` watches that metric. The condition fires when the `DELTA SUM` over a 600-second window is **> 0** (i.e., any single ERROR log triggers it immediately, `duration="0s"`).
    
4. **PagerDuty notification** — The alert policy's `notification_channels` points to the `google_monitoring_notification_channel` resource configured with a PagerDuty service key (`var.pagerduty_channel_service_key`), which fires the page you saw.
